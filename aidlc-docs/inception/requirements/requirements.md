# 테이블오더 서비스 - 요구사항 문서

## Intent Analysis

| 항목 | 내용 |
|---|---|
| **User Request** | 디지털 테이블오더 서비스 구축 |
| **Request Type** | New Project (Greenfield) |
| **Scope** | System-wide (고객 UI + 관리자 UI + 백엔드 + DB) |
| **Complexity** | Complex (다중 사용자, 실시간 통신, 멀티테넌트) |

---

## 1. 기술 스택

| 영역 | 기술 |
|---|---|
| **백엔드** | Python + FastAPI |
| **프론트엔드** | Next.js (TypeScript) |
| **데이터베이스** | PostgreSQL |
| **실시간 통신** | Server-Sent Events (SSE) |
| **인증** | JWT 토큰 기반 |
| **배포** | Docker 컨테이너 |

---

## 2. 아키텍처 결정

- **UI 구조**: 단일 Next.js 앱에서 경로(route)로 고객용/관리자용 분리
- **매장 구조**: 다중 매장 지원 (멀티테넌트)
- **관리자 계정**: 매장당 여러 관리자 계정 (역할 구분 없음)
- **카테고리 관리**: 관리자가 CRUD 가능
- **이미지 관리**: 서버에 이미지 파일 업로드 후 URL 자동 생성
- **예상 규모**: 중규모 (테이블 10~50개, 동시 주문 10~30건)
- **보안 확장**: 미적용 (프로토타입 수준)

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 1회 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 별도 로그인 절차 없이 즉시 주문 가능

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제
- 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인
- 주문 확정 버튼
- 성공 시: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트 → 장바구니 자동 비우기
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- 매장 이용 완료 처리된 주문 제외

### 3.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트
- 그리드/대시보드 레이아웃 (테이블별 카드 형태)
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 2초 이내 주문 표시
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산 → 피드백
- **테이블 세션 관리**:
  - 세션 시작: 첫 주문 시 자동 시작
  - 세션 종료(이용 완료): 확인 팝업 → 주문 내역을 과거 이력으로 이동 → 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지)
- 이미지 파일 업로드 → 서버 저장 → URL 자동 생성
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

#### FR-A05: 카테고리 관리
- 카테고리 CRUD (추가/수정/삭제)
- 카테고리 표시 순서 관리

### 3.3 매장 관리 (Store Management)

#### FR-S01: 매장 등록 및 관리
- 다중 매장 지원 (멀티테넌트)
- 매장별 독립 데이터 관리 (메뉴, 테이블, 주문, 관리자)
- 매장 식별자 기반 데이터 격리

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE를 통한 실시간 주문 업데이트 (2초 이내)
- 중규모 동시 접속 지원 (테이블 10~50개, 동시 주문 10~30건)

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 터치 타겟)
- 직관적 카드 기반 레이아웃
- 명확한 시각적 계층 구조

### NFR-03: 데이터 관리
- 장바구니 클라이언트 측 로컬 저장 (새로고침 유지)
- 주문 이력 DB 저장 (OrderHistory)
- 세션 ID로 주문 그룹화

### NFR-04: 배포
- Docker 컨테이너 기반 배포
- 프론트엔드/백엔드/DB 컨테이너 분리

## 5. 제외 범위 (Out of Scope)

요구사항 constraints.md에 명시된 항목:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도/번역 API)

## 6. 용어 정의

| 용어 | 정의 |
|---|---|
| **테이블 세션** | 고객이 첫 주문 시작 ~ 매장 이용 완료 처리까지의 기간. 종료 후 새 고객의 첫 주문 시 새 세션 시작 |
| **매장 식별자** | 각 매장을 구분하는 고유 ID |
| **멀티테넌트** | 하나의 시스템에서 여러 매장이 독립적으로 데이터를 관리하는 구조 |
