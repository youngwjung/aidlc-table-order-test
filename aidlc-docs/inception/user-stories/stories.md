# 유저 스토리 (User Stories) - 테이블오더 서비스

**분류 방식**: Persona-Based
**세분화 수준**: 세밀 (개별 동작 단위)
**AC 형식**: Given-When-Then (BDD)
**우선순위**: MoSCoW (Must/Should/Could/Won't)

---

# 고객 스토리 (Customer Stories)

**Persona**: 김민수 (매장 고객)

---

## CS-01: 태블릿 자동 로그인
**Priority**: Must

**As a** 고객, **I want** 테이블 태블릿에서 별도 로그인 없이 자동으로 인증되기를, **so that** 즉시 메뉴를 탐색하고 주문할 수 있다.

### Acceptance Criteria
- **Given** 관리자가 태블릿 초기 설정을 완료한 상태일 때, **When** 태블릿 브라우저에서 서비스에 접근하면, **Then** 저장된 인증 정보로 자동 로그인되어 메뉴 화면이 표시된다.
- **Given** 자동 로그인에 성공한 상태일 때, **When** 브라우저를 새로고침하면, **Then** 로컬에 저장된 정보로 다시 자동 로그인된다.

---

## CS-02: 카테고리별 메뉴 목록 조회
**Priority**: Must

**As a** 고객, **I want** 카테고리별로 분류된 메뉴 목록을 볼 수 있기를, **so that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다.

### Acceptance Criteria
- **Given** 메뉴 화면에 진입한 상태일 때, **When** 화면이 로드되면, **Then** 모든 카테고리와 각 카테고리에 속한 메뉴가 카드 형태로 표시된다.
- **Given** 메뉴 목록이 표시된 상태일 때, **When** 특정 카테고리를 선택하면, **Then** 해당 카테고리의 메뉴만 필터링되어 표시된다.

---

## CS-03: 카테고리 간 빠른 이동
**Priority**: Must

**As a** 고객, **I want** 카테고리 간에 빠르게 이동할 수 있기를, **so that** 다양한 카테고리의 메뉴를 편리하게 탐색할 수 있다.

### Acceptance Criteria
- **Given** 메뉴 화면에 있는 상태일 때, **When** 카테고리 탭/버튼을 터치하면, **Then** 해당 카테고리 섹션으로 즉시 스크롤 이동한다.
- **Given** 카테고리 목록이 표시된 상태일 때, **When** 현재 선택된 카테고리가 있으면, **Then** 해당 카테고리가 시각적으로 강조 표시된다.

---

## CS-04: 메뉴 상세 정보 확인
**Priority**: Must

**As a** 고객, **I want** 메뉴의 상세 정보(메뉴명, 가격, 설명, 이미지)를 확인할 수 있기를, **so that** 주문 전에 충분한 정보를 바탕으로 선택할 수 있다.

### Acceptance Criteria
- **Given** 메뉴 목록이 표시된 상태일 때, **When** 메뉴 카드를 터치하면, **Then** 메뉴명, 가격, 설명, 이미지가 포함된 상세 정보가 표시된다.
- **Given** 메뉴 상세 화면이 표시된 상태일 때, **When** 이미지가 등록되지 않은 메뉴이면, **Then** 기본 플레이스홀더 이미지가 표시된다.

---

## CS-05: 장바구니에 메뉴 추가
**Priority**: Must

**As a** 고객, **I want** 원하는 메뉴를 장바구니에 추가할 수 있기를, **so that** 주문할 메뉴를 미리 선택해 둘 수 있다.

### Acceptance Criteria
- **Given** 메뉴 목록 또는 상세 화면에 있을 때, **When** 메뉴의 "담기" 버튼을 터치하면, **Then** 해당 메뉴가 수량 1로 장바구니에 추가된다.
- **Given** 이미 장바구니에 동일 메뉴가 있을 때, **When** 같은 메뉴를 다시 추가하면, **Then** 해당 메뉴의 수량이 1 증가한다.

---

## CS-06: 장바구니 메뉴 수량 조절
**Priority**: Must

**As a** 고객, **I want** 장바구니에 담긴 메뉴의 수량을 증가/감소시킬 수 있기를, **so that** 주문 전에 원하는 수량으로 조정할 수 있다.

### Acceptance Criteria
- **Given** 장바구니에 메뉴가 있을 때, **When** 수량 증가(+) 버튼을 터치하면, **Then** 해당 메뉴의 수량이 1 증가하고 총 금액이 재계산된다.
- **Given** 장바구니에 수량 2 이상인 메뉴가 있을 때, **When** 수량 감소(-) 버튼을 터치하면, **Then** 해당 메뉴의 수량이 1 감소하고 총 금액이 재계산된다.
- **Given** 장바구니에 수량 1인 메뉴가 있을 때, **When** 수량 감소(-) 버튼을 터치하면, **Then** 해당 메뉴가 장바구니에서 삭제된다.

---

## CS-07: 장바구니 메뉴 삭제
**Priority**: Must

**As a** 고객, **I want** 장바구니에서 특정 메뉴를 삭제할 수 있기를, **so that** 주문하지 않을 메뉴를 제거할 수 있다.

### Acceptance Criteria
- **Given** 장바구니에 메뉴가 있을 때, **When** 메뉴의 삭제 버튼을 터치하면, **Then** 해당 메뉴가 장바구니에서 제거되고 총 금액이 재계산된다.

---

## CS-08: 장바구니 비우기
**Priority**: Should

**As a** 고객, **I want** 장바구니를 한 번에 비울 수 있기를, **so that** 처음부터 다시 메뉴를 선택할 수 있다.

### Acceptance Criteria
- **Given** 장바구니에 1개 이상의 메뉴가 있을 때, **When** "장바구니 비우기" 버튼을 터치하면, **Then** 모든 메뉴가 제거되고 총 금액이 0원으로 표시된다.

---

## CS-09: 장바구니 총 금액 실시간 확인
**Priority**: Must

**As a** 고객, **I want** 장바구니의 총 금액을 실시간으로 확인할 수 있기를, **so that** 주문 금액을 미리 파악할 수 있다.

### Acceptance Criteria
- **Given** 장바구니에 메뉴가 추가/삭제/수량 변경될 때, **When** 변경이 발생하면, **Then** 총 금액이 즉시 재계산되어 표시된다.
- **Given** 장바구니가 비어 있을 때, **When** 장바구니 화면을 보면, **Then** 총 금액이 0원으로 표시된다.

---

## CS-10: 장바구니 새로고침 유지
**Priority**: Must

**As a** 고객, **I want** 브라우저를 새로고침해도 장바구니가 유지되기를, **so that** 실수로 새로고침하더라도 선택한 메뉴를 잃지 않는다.

### Acceptance Criteria
- **Given** 장바구니에 메뉴가 담겨 있을 때, **When** 브라우저를 새로고침하면, **Then** 장바구니의 메뉴, 수량, 총 금액이 이전 상태 그대로 복원된다.

---

## CS-11: 주문 내역 확인 및 확정
**Priority**: Must

**As a** 고객, **I want** 주문 전에 최종 주문 내역을 확인하고 확정할 수 있기를, **so that** 실수 없이 정확한 주문을 할 수 있다.

### Acceptance Criteria
- **Given** 장바구니에 메뉴가 있을 때, **When** "주문하기" 버튼을 터치하면, **Then** 주문 확인 화면이 표시되며 메뉴명, 수량, 단가, 총 금액이 나열된다.
- **Given** 주문 확인 화면에서, **When** "주문 확정" 버튼을 터치하면, **Then** 주문이 서버로 전송된다.

---

## CS-12: 주문 성공 확인 및 리다이렉트
**Priority**: Must

**As a** 고객, **I want** 주문 성공 시 주문 번호를 확인한 뒤 자동으로 메뉴 화면으로 돌아가기를, **so that** 추가 주문을 바로 할 수 있다.

### Acceptance Criteria
- **Given** 주문이 서버에 성공적으로 전송된 상태일 때, **When** 성공 응답을 받으면, **Then** 주문 번호가 화면에 표시된다.
- **Given** 주문 성공 화면이 표시된 상태일 때, **When** 5초가 경과하면, **Then** 자동으로 메뉴 화면으로 리다이렉트된다.
- **Given** 주문이 성공한 상태일 때, **When** 메뉴 화면으로 돌아가면, **Then** 장바구니가 자동으로 비워진다.

---

## CS-13: 주문 내역 조회
**Priority**: Must

**As a** 고객, **I want** 현재 테이블 세션의 주문 내역을 조회할 수 있기를, **so that** 이번 방문에서 주문한 내용을 확인할 수 있다.

### Acceptance Criteria
- **Given** 주문 내역 화면에 진입한 상태일 때, **When** 화면이 로드되면, **Then** 현재 테이블 세션의 주문만 시간순으로 표시된다.
- **Given** 주문 내역이 표시된 상태일 때, **When** 각 주문을 확인하면, **Then** 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다.
- **Given** 테이블 세션이 종료(이용 완료)된 주문이 있을 때, **When** 주문 내역을 조회하면, **Then** 이전 세션 주문은 표시되지 않는다.

---

# 고객 에러 스토리 (Customer Error Stories)

---

## CS-ERR-01: 자동 로그인 실패 처리
**Priority**: Must

**As a** 고객, **I want** 자동 로그인 실패 시 안내 메시지를 볼 수 있기를, **so that** 문제 상황을 인지하고 직원에게 도움을 요청할 수 있다.

### Acceptance Criteria
- **Given** 저장된 인증 정보가 없거나 유효하지 않을 때, **When** 서비스에 접근하면, **Then** "관리자에게 테이블 설정을 요청해 주세요" 안내 메시지가 표시된다.
- **Given** 서버와 통신할 수 없는 상태일 때, **When** 자동 로그인을 시도하면, **Then** "네트워크 연결을 확인해 주세요" 에러 메시지가 표시된다.

---

## CS-ERR-02: 주문 전송 실패 처리
**Priority**: Must

**As a** 고객, **I want** 주문 전송 실패 시 에러 메시지를 확인하고 재시도할 수 있기를, **so that** 주문을 완료할 수 있다.

### Acceptance Criteria
- **Given** 주문 확정 버튼을 눌렀을 때, **When** 서버 오류 등으로 주문 전송이 실패하면, **Then** 에러 메시지가 표시되고 장바구니가 유지된다.
- **Given** 주문 전송 실패 상태일 때, **When** "다시 시도" 버튼을 터치하면, **Then** 주문이 재전송된다.

---

## CS-ERR-03: 메뉴 로드 실패 처리
**Priority**: Should

**As a** 고객, **I want** 메뉴 로드 실패 시 안내를 받을 수 있기를, **so that** 문제 상황을 인지할 수 있다.

### Acceptance Criteria
- **Given** 메뉴 화면에 진입한 상태일 때, **When** 서버에서 메뉴 데이터를 가져오지 못하면, **Then** "메뉴를 불러올 수 없습니다. 잠시 후 다시 시도해 주세요" 에러 메시지와 "새로고침" 버튼이 표시된다.

---

## CS-ERR-04: 빈 장바구니 주문 방지
**Priority**: Must

**As a** 고객, **I want** 장바구니가 비어 있을 때 주문이 불가능하기를, **so that** 빈 주문이 실수로 접수되지 않는다.

### Acceptance Criteria
- **Given** 장바구니가 비어 있는 상태일 때, **When** "주문하기" 버튼을 확인하면, **Then** 버튼이 비활성화 상태이다.

---

# 관리자 스토리 (Admin Stories)

**Persona**: 박서연 (매장 관리자)

---

## AD-01: 관리자 로그인
**Priority**: Must

**As a** 관리자, **I want** 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있기를, **so that** 매장 관리 시스템에 접근할 수 있다.

### Acceptance Criteria
- **Given** 로그인 화면에서, **When** 올바른 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인 버튼을 누르면, **Then** JWT 토큰이 발급되고 주문 모니터링 대시보드로 이동한다.
- **Given** 로그인에 성공한 상태일 때, **When** 16시간 이내에 브라우저를 새로고침하면, **Then** 세션이 유지되어 재로그인 없이 사용 가능하다.

---

## AD-02: 관리자 자동 로그아웃
**Priority**: Must

**As a** 관리자, **I want** 16시간 후 자동으로 로그아웃되기를, **so that** 보안이 유지된다.

### Acceptance Criteria
- **Given** 로그인한 지 16시간이 경과한 상태일 때, **When** 서비스에 접근하면, **Then** 자동으로 로그아웃되고 로그인 화면으로 리다이렉트된다.

---

## AD-03: 실시간 주문 대시보드 조회
**Priority**: Must

**As a** 관리자, **I want** 테이블별 주문 현황을 그리드 레이아웃으로 한눈에 볼 수 있기를, **so that** 매장 전체 주문 상황을 파악할 수 있다.

### Acceptance Criteria
- **Given** 주문 모니터링 대시보드에 진입한 상태일 때, **When** 화면이 로드되면, **Then** 각 테이블이 카드 형태로 그리드 레이아웃에 표시된다.
- **Given** 테이블 카드가 표시된 상태일 때, **When** 각 카드를 확인하면, **Then** 테이블 번호, 총 주문액, 최신 주문 n개 미리보기가 표시된다.

---

## AD-04: 실시간 주문 수신 (SSE)
**Priority**: Must

**As a** 관리자, **I want** 새로운 주문이 들어오면 실시간으로 대시보드에 반영되기를, **so that** 주문을 즉시 확인하고 처리할 수 있다.

### Acceptance Criteria
- **Given** 대시보드가 SSE로 연결된 상태일 때, **When** 고객이 새 주문을 생성하면, **Then** 2초 이내에 해당 테이블 카드에 주문이 표시된다.
- **Given** 신규 주문이 수신된 상태일 때, **When** 대시보드에 반영되면, **Then** 해당 주문이 시각적으로 강조(색상 변경, 애니메이션)된다.

---

## AD-05: 주문 상세 보기
**Priority**: Must

**As a** 관리자, **I want** 테이블 카드를 클릭하여 전체 주문 목록을 상세히 볼 수 있기를, **so that** 각 주문의 정확한 메뉴와 수량을 확인할 수 있다.

### Acceptance Criteria
- **Given** 대시보드에서 테이블 카드가 표시된 상태일 때, **When** 테이블 카드를 클릭하면, **Then** 해당 테이블의 전체 주문 목록(주문 번호, 시각, 메뉴/수량, 금액, 상태)이 상세 보기로 표시된다.

---

## AD-06: 주문 상태 변경
**Priority**: Must

**As a** 관리자, **I want** 주문 상태를 대기중 → 준비중 → 완료로 변경할 수 있기를, **so that** 주문 처리 진행 상황을 관리할 수 있다.

### Acceptance Criteria

- **Given** "준비중" 상태의 주문이 있을 때, **When** 상태 변경 버튼을 클릭하면, **Then** 주문 상태가 "완료"로 변경된다.

- **Given** SSE 연결이 끊어진 상태일 때, **When** 재연결이 시도되면, **Then** 자동으로 재연결되고 끊긴 동안의 주문이 동기화된다.
