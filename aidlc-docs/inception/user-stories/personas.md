# 사용자 페르소나 (User Personas)

---

## Persona 1: 매장 고객 (Customer)

| 항목 | 내용 |
|---|---|
| **이름** | 김민수 (대표 고객) |
| **역할** | 매장 방문 고객 |
| **연령대** | 20~50대 |
| **기술 수준** | 다양함 (스마트폰/태블릿 기본 사용 가능) |
| **사용 환경** | 매장 테이블 태블릿 (웹 브라우저) |

### 목표 (Goals)
- 대기 없이 빠르게 메뉴를 주문하고 싶다
- 메뉴를 쉽게 탐색하고 상세 정보를 확인하고 싶다
- 주문 전에 선택한 메뉴를 검토하고 수정하고 싶다
- 주문 상태를 실시간으로 확인하고 싶다

### 불편 사항 (Pain Points)
- 직원을 불러 주문하는 과정에서 대기 시간 발생
- 메뉴판이 복잡하거나 정보가 부족한 경우
- 주문이 정확히 전달되었는지 불안한 경우
- 이전에 뭘 주문했는지 기억이 안 나는 경우

### 사용 빈도
- 매장 방문 시마다 (비정기적)
- 한 번의 방문에 여러 차례 추가 주문 가능

### 주요 관련 스토리
- CS-01 ~ CS-13 (고객 기본 기능)
- CS-ERR-01 ~ CS-ERR-04 (고객 에러 시나리오)

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 박서연 (대표 관리자) |
| **역할** | 매장 운영자 / 점장 / 직원 |
| **연령대** | 25~55대 |
| **기술 수준** | 중간 (웹 서비스 기본 사용 가능, IT 전문가 아님) |
| **사용 환경** | PC 또는 태블릿 (웹 브라우저) |

### 목표 (Goals)
- 실시간으로 주문을 모니터링하고 효율적으로 처리하고 싶다
- 메뉴와 카테고리를 쉽게 관리하고 싶다
- 테이블별 주문 상황을 한눈에 파악하고 싶다
- 과거 주문 내역을 조회하고 운영에 활용하고 싶다

### 불편 사항 (Pain Points)
- 수기 주문 접수 시 누락이나 오류 발생
- 바쁜 시간대에 주문 현황 파악이 어려움
- 메뉴 변경 시 모든 테이블 메뉴판 교체 필요
- 일일 정산 시 주문 내역 확인이 번거로움

### 사용 빈도
- 영업 시간 내 지속적 사용 (매일)
- 메뉴 관리는 비정기적

### 주요 관련 스토리
- AD-01 ~ AD-22 (관리자 기본 기능)
- AD-ERR-01 ~ AD-ERR-05 (관리자 에러 시나리오)
