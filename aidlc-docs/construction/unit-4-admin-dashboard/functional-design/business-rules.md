# 비즈니스 규칙 - Unit 4: Admin Dashboard & Table Management

---

## 1. 주문 상태 전이 규칙

### BR-ORD-01: 순방향 상태 전이만 허용
- **규칙**: 주문 상태는 `pending → preparing → completed` 순서로만 변경 가능
- **역방향 금지**: preparing → pending, completed → preparing/pending 불가
- **단계 건너뛰기 금지**: pending → completed 직접 변경 불가
- **위반 시**: 400 Bad Request, "현재 상태에서 해당 상태로 변경할 수 없습니다"

### BR-ORD-02: 주문 삭제 정책
- **규칙**: 관리자는 모든 상태(pending, preparing, completed)의 주문을 삭제 가능
- **확인**: 삭제 전 반드시 확인 팝업 표시
- **효과**: 주문 삭제 시 해당 테이블의 총 주문액 자동 재계산
- **실패 처리**: 삭제 실패 시 에러 메시지 표시, 주문 데이터 유지

### BR-ORD-03: 주문 번호 불변성
- **규칙**: 주문 번호는 생성 후 변경 불가
- **스냅샷**: OrderItem의 menu_name, unit_price는 주문 시점 값 유지

---

## 2. 테이블 세션 관리 규칙

### BR-TBL-01: 세션 시작
- **규칙**: 테이블 세션은 첫 주문 시 자동 시작 (Unit 3에서 처리)
- **조건**: session_id가 NULL인 테이블에 주문 생성 시
- **효과**: UUID v4로 session_id 생성, session_started_at 설정

### BR-TBL-02: 이용 완료 (세션 종료)
- **규칙**: 관리자만 이용 완료 처리 가능
- **전제**: 활성 세션(session_id != NULL)이 존재해야 함
- **미완료 주문 처리**: 미완료 주문이 있어도 경고 후 강제 진행 가능
- **효과**:
  1. 현재 세션의 모든 주문 → OrderHistory로 이동
  2. session_id = NULL, session_started_at = NULL로 리셋
  3. SSE `table_completed` 이벤트 브로드캐스트

### BR-TBL-03: 테이블 삭제 제약
- **규칙**: 활성 세션이 있는 테이블은 삭제 불가
- **규칙**: 현재 주문이 있는 테이블은 삭제 불가
- **해결**: 이용 완료 처리 후 삭제 가능

### BR-TBL-04: 테이블 번호 유니크
- **규칙**: 동일 매장 내 테이블 번호 중복 불가
- **범위**: (store_id, table_number) 복합 유니크 제약

### BR-TBL-05: 테이블 비밀번호
- **규칙**: 비밀번호는 평문 저장 (태블릿 초기 설정용)
- **용도**: 고객 태블릿 자동 로그인에 사용

## 3. SSE 연결 규칙

### BR-SSE-01: 구독 범위
- **규칙**: store_id 기반 구독, 해당 매장의 이벤트만 수신
- **인증**: admin 역할의 JWT 토큰 필수

### BR-SSE-02: 재연결 전략
- **방식**: 지수 백오프 (1s → 2s → 4s → 8s → 16s → 30s 최대)
- **재연결 성공 시**: 전체 주문 데이터 재동기화 (GET /orders)
- **UI**: 연결 끊김 시 알림 배너 표시

### BR-SSE-03: Heartbeat
- **규칙**: 30초마다 ping 이벤트 전송
- **목적**: 연결 유지, 프록시/로드밸런서 타임아웃 방지

### BR-SSE-04: 알림
- **규칙**: 신규 주문 수신 시 브라우저 알림음(beep) 재생
- **시각적 강조**: 신규 주문 카드 하이라이트 애니메이션

---

## 4. 과거 주문 이력 규칙

### BR-HIST-01: 이력 생성
- **규칙**: 이용 완료 시에만 주문이 이력으로 이동
- **데이터**: Order → OrderHistory, OrderItem → OrderHistoryItem 복사
- **스냅샷**: table_number는 이동 시점 값 스냅샷 (테이블 삭제/변경 후에도 유지)

### BR-HIST-02: 이력 조회 기본 범위
- **규칙**: 기본 최근 7일 조회
- **필터**: 시작일/종료일 날짜 범위 선택 가능
- **정렬**: completed_at 내림차순 (최신 우선)

### BR-HIST-03: 이력 불변성
- **규칙**: 이력 데이터는 수정/삭제 불가 (읽기 전용)

---

## 5. 멀티테넌트 격리 규칙

### BR-MT-01: store_id 기반 격리
- **규칙**: 모든 API 쿼리에 store_id 조건 필수
- **검증**: JWT의 store_id와 URL 경로의 store_id 일치 확인 (미들웨어)
- **불일치 시**: 403 Forbidden

### BR-MT-02: SSE 격리
- **규칙**: SSE 이벤트는 동일 store_id의 구독자에게만 전달
- **구독 키**: store_id

---

## 6. 대시보드 표시 규칙

### BR-DASH-01: 테이블 카드 미리보기
- **규칙**: 각 테이블 카드에 최신 주문 3개 미리보기 표시
- **정렬**: created_at DESC
- **내용**: 주문번호, 상태 배지, 금액

### BR-DASH-02: 총 주문액
- **규칙**: 현재 세션의 모든 활성 주문 금액 합계
- **삭제된 주문 제외**: 삭제된 주문은 합계에서 자동 차감
- **이용 완료 시**: 총 주문액 0으로 리셋

### BR-DASH-03: 실시간 업데이트
- **규칙**: SSE 이벤트 수신 시 해당 테이블 카드 즉시 업데이트
- **신규 주문**: 카드 하이라이트 + 알림음
- **상태 변경**: 상태 배지 업데이트
- **주문 삭제**: 주문 목록 및 총액 업데이트
- **이용 완료**: 테이블 카드 "빈 테이블" 상태로 전환

---

## 7. 입력 검증 규칙

### BR-VAL-01: 테이블 번호
- **타입**: 양의 정수
- **범위**: 1 ~ 999
- **필수**: Yes

### BR-VAL-02: 테이블 비밀번호
- **타입**: 문자열
- **길이**: 1 ~ 50자
- **필수**: Yes

### BR-VAL-03: 주문 상태 값
- **허용값**: "pending", "preparing", "completed"
- **기타 값**: 422 Unprocessable Entity

### BR-VAL-04: 날짜 범위
- **형식**: YYYY-MM-DD
- **제약**: date_from <= date_to
- **미래 날짜**: 허용 (오늘 이후도 조회 가능)
