# 비기능 요구사항 - Unit 4: Admin Dashboard & Table Management

> Unit 1 Foundation의 NFR을 상속하며, Unit 4 특화 요구사항을 추가 정의합니다.

---

## 1. 성능 (Performance)

| 항목 | 목표 | 비고 |
|---|---|---|
| SSE 이벤트 전달 | 2초 이내 | 주문 생성/상태변경 → 대시보드 반영 |
| SSE 연결 수립 | 2초 이내 | EventSource 초기 연결 |
| 대시보드 초기 로드 | 3초 이내 | 테이블 목록 + 전체 주문 데이터 |
| 주문 상태 변경 API | 500ms 이내 | PUT /orders/{id}/status |
| 주문 삭제 API | 500ms 이내 | DELETE /orders/{id} |
| 이용 완료 API | 1초 이내 | 주문 이력 이동 트랜잭션 포함 |
| 과거 내역 조회 | 1초 이내 | 7일 기본 범위 기준 |
| SSE Heartbeat | 30초 간격 | 연결 유지 |

### SSE 성능 고려사항
- 인메모리 asyncio.Queue 사용으로 오버헤드 최소화
- 이벤트 데이터는 필요한 필드만 포함 (전체 Order 객체 대신 변경 데이터만)
- Heartbeat ping으로 프록시/로드밸런서 타임아웃 방지

## 2. 확장성 (Scalability)

| 항목 | 목표 |
|---|---|
| 동시 SSE 연결 | 50개 이하 (매장당) |
| 테이블 수 | 매장당 최대 999개 |
| 주문 이력 보존 | 제한 없음 |
| 이벤트 큐 크기 | 구독자당 100개 이벤트 버퍼 |

### SSE 확장 제약
- 단일 프로세스 인메모리 구독 (수평 확장 미고려)
- 프로토타입 단계에서는 단일 서버로 충분

## 3. 가용성 (Availability)

| 항목 | 구현 |
|---|---|
| SSE 재연결 | 지수 백오프 (1s~30s) |
| 데이터 동기화 | 재연결 시 전체 주문 재조회 |
| 연결 상태 표시 | 대시보드에 연결 끊김 배너 |

---

## 4. 보안 (Security)

| 항목 | 구현 |
|---|---|
| SSE 인증 | JWT 토큰 (query parameter) |
| SSE 권한 | admin 역할만 구독 가능 |
| store_id 격리 | JWT store_id와 경로 store_id 일치 검증 |
| 주문 삭제 권한 | admin 역할만 가능 |
| 이용 완료 권한 | admin 역할만 가능 |

---

## 5. 사용성 (Usability)

| 항목 | 구현 |
|---|---|
| 신규 주문 알림 | 시각적 강조 + 알림음 (beep) |
| 확인 팝업 | 삭제, 이용 완료 시 ConfirmModal |
| 상태 변경 피드백 | 즉각적 UI 업데이트 + Toast |
| 연결 상태 | SSE 끊김 시 배너 알림 |
| 대시보드 필터 | 테이블별 클라이언트 필터링 |
| 반응형 | 데스크톱/태블릿 최적화 (그리드 자동 조정) |

---

## 6. 신뢰성 (Reliability)

| 항목 | 구현 |
|---|---|
| 이벤트 유실 복구 | SSE 재연결 시 전체 데이터 재조회 |
| 이용 완료 트랜잭션 | 주문 이력 이동 + 세션 리셋 원자적 처리 |
| 주문 삭제 CASCADE | OrderItem 자동 삭제 (DB 레벨) |
| 에러 복구 | 삭제/상태변경 실패 시 원본 데이터 유지 |
