# 비즈니스 규칙 - Unit 2: Menu & Category Management

---

## 1. 카테고리 규칙

### BR-CAT01: 카테고리명 규칙
- 필수, 1~50자
- 공백 허용, 빈 문자열 불허
- 매장 내 고유 (store_id + name UNIQUE)

### BR-CAT02: 카테고리 삭제 제한
- 해당 카테고리에 메뉴가 1개 이상 존재하면 삭제 불가
- 에러 메시지: "해당 카테고리에 메뉴가 있어 삭제할 수 없습니다"
- 메뉴를 모두 삭제하거나 다른 카테고리로 이동 후 삭제 가능

### BR-CAT03: 카테고리 표시 순서
- display_order: 0 이상 정수
- 기본값: 0
- 순서 변경 API: category_ids 배열 순서로 display_order 재설정 (0, 1, 2, ...)
- 고객/관리자 화면 모두 display_order ASC 순으로 표시

### BR-CAT04: 카테고리 멀티테넌트 격리
- 모든 카테고리 조회/생성/수정/삭제에 store_id 조건 필수
- JWT의 store_id와 URL 경로의 store_id 일치 검증

## 2. 메뉴 규칙

### BR-MENU01: 메뉴 필수 필드
- name: 필수, 1~100자
- price: 필수, 0 이상 10,000,000 이하 정수 (원 단위)
- category_id: 필수, 해당 매장의 유효한 카테고리 ID

### BR-MENU02: 메뉴 선택 필드
- description: 선택, Text
- image_url: 선택, 이미지 업로드 시 자동 생성
- display_order: 선택, 0 이상 정수, 기본값 0

### BR-MENU03: 메뉴 가격 검증
- 0 이상 정수만 허용 (음수 불가)
- 최대값: 10,000,000 (천만 원)
- 문자, 소수점, 음수 입력 시 검증 에러

### BR-MENU04: 메뉴 카테고리 참조 무결성
- 메뉴 생성/수정 시 category_id가 해당 store_id의 유효한 카테고리인지 확인
- 존재하지 않는 카테고리 참조 시 400 에러

### BR-MENU05: 메뉴 표시 순서
- display_order: 0 이상 정수
- 기본값: 0
- 순서 변경 API: menu_ids 배열 순서로 display_order 재설정
- 카테고리 내에서의 순서를 의미

### BR-MENU06: 메뉴 삭제 시 이미지 정리
- 메뉴에 연결된 이미지가 있으면 파일 시스템에서 함께 삭제
- 이미지 삭제 실패 시에도 메뉴 삭제는 진행 (파일 정리는 best-effort)

### BR-MENU07: 메뉴 멀티테넌트 격리
- 모든 메뉴 조회/생성/수정/삭제에 store_id 조건 필수
- JWT의 store_id와 URL 경로의 store_id 일치 검증

## 3. 이미지 규칙

### BR-IMG01: 허용 파일 형식
- JPEG (image/jpeg)
- PNG (image/png)
- WebP (image/webp)
- 그 외 형식 -> 400 "JPEG, PNG, WebP 형식만 허용됩니다"

### BR-IMG02: 파일 크기 제한
- 최대 5MB (5,242,880 bytes)
- 초과 시 -> 400 "파일 크기는 5MB 이하여야 합니다"

### BR-IMG03: 파일명 생성
- UUID v4 + 원본 확장자
- 형식: `{uuid}.{ext}` (예: "a1b2c3d4-e5f6-7890-abcd-ef1234567890.jpg")
- 파일명 충돌 방지

### BR-IMG04: 이미지 저장 경로
- 저장 디렉토리: `uploads/`
- URL 경로: `/uploads/{filename}`
- Docker 볼륨으로 영속화

### BR-IMG05: 이미지 리사이징
- 리사이징 없음 - 원본 그대로 저장 (MVP 단계)

### BR-IMG06: 이미지 없는 메뉴
- 이미지 미등록 메뉴는 image_url = NULL
- 고객 화면에서 기본 플레이스홀더 이미지 표시 (프론트엔드 처리)

### BR-IMG07: 이미지 교체
- 메뉴 수정 시 새 이미지 업로드하면 기존 이미지 파일 삭제
- 새 이미지 파일 저장 후 image_url 업데이트

## 4. 프론트엔드 검증 규칙

### BR-FV01: 메뉴 등록/수정 폼 검증
- 메뉴명: 필수, 빈 값 시 "메뉴명을 입력해 주세요"
- 가격: 필수, 숫자만, 0 이상, "유효한 가격을 입력해 주세요"
- 카테고리: 필수, "카테고리를 선택해 주세요"
- 이미지: 선택, 형식/크기 검증은 서버에서 수행

### BR-FV02: 카테고리 등록/수정 폼 검증
- 카테고리명: 필수, 빈 값 시 "카테고리명을 입력해 주세요"

### BR-FV03: 폼 에러 표시
- 필수 필드 미입력 시 해당 필드 아래에 빨간색 에러 메시지
- 서버 에러 시 Toast로 에러 메시지 표시
- 폼 입력 내용은 에러 시에도 유지
