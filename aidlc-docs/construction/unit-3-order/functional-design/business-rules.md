# Unit 3: Customer Order Flow - Business Rules

## Overview
Unit 3에서 적용되는 비즈니스 규칙을 정의합니다. Unit 1에서 정의된 공통 규칙(BR-V01~V03, BR-TS01~TS03, BR-ON01, BR-MT01~MT02)을 기반으로 합니다.

---

## 1. 장바구니 규칙 (BR-C01 ~ BR-C08)

### BR-C01: 메뉴 추가
- 장바구니에 메뉴를 추가할 때 menuId, menuName, price, quantity, imageUrl을 저장
- 이미 동일 menuId가 장바구니에 있으면 수량을 합산 (새 항목 생성하지 않음)

### BR-C02: 수량 제한
- 최소 수량: 1
- 최대 수량: 99 (BR-V02 준수)
- 수량이 0이 되면 해당 항목 자동 삭제

### BR-C03: 수량 변경
- +/- 버튼으로 1씩 증감
- - 버튼으로 수량이 0이 되면 항목 자동 삭제 (별도 확인 없음)
- + 버튼은 최대 수량(99)까지만 허용

### BR-C04: 항목 삭제
- 개별 항목 삭제 가능 (스와이프 또는 삭제 버튼)
- 수량 0 도달 시 자동 삭제 (BR-C02)

### BR-C05: 장바구니 전체 초기화
- "전체 삭제" 버튼으로 장바구니 비우기
- 확인 모달 표시 후 실행

### BR-C06: 합계 금액 계산
- 합계 = Σ(item.price × item.quantity) for all items
- 실시간 갱신 (항목 추가/수정/삭제 시 즉시 반영)
- BR-V03 준수: 서버 제출 시에도 동일 로직으로 검증

### BR-C07: localStorage 영속성
- 장바구니 변경 시마다 localStorage에 동기화
- 키 형식: `cart_{storeId}_{tableId}`
- 브라우저 새로고침 시 localStorage에서 복원
- JSON 직렬화/역직렬화

### BR-C08: 빈 장바구니 주문 방지
- 장바구니가 비어있으면 "주문하기" 버튼 비활성화
- CS-ERR-04 스토리 구현

---

## 2. 주문 생성 규칙 (BR-O01 ~ BR-O06)

### BR-O01: 주문 제출 전 확인
- "주문하기" 버튼 클릭 시 확인 모달 표시
- 모달에 주문 항목 요약, 총 금액 표시
- "확인" 클릭 시 서버로 주문 제출

### BR-O02: 서버 메뉴 유효성 검증
- 주문 제출 시 각 항목의 menu_id로 Menu 테이블 조회
- 메뉴가 존재하지 않으면 해당 항목 오류 반환
- 메뉴가 존재하면 **서버의 현재 가격**으로 total_amount 재계산
- 클라이언트가 보낸 가격과 서버 가격이 다르면 서버 가격 적용 (스냅샷)

### BR-O03: 주문번호 생성
- BR-ON01 준수: 매장별 일일 순번
- 당일 해당 매장의 주문 수 + 1
- 3자리 문자열 형식 ("001", "042", "999")

### BR-O04: 초기 주문 상태
- 주문 생성 시 status = "waiting"
- status 변경은 Unit 4 (Admin) 책임

### BR-O05: OrderItem 스냅샷
- 주문 시점의 메뉴명(menu_name)과 단가(unit_price)를 OrderItem에 스냅샷 저장
- 이후 메뉴 가격이 변경되어도 주문 기록은 영향받지 않음

### BR-O06: 멀티테넌트 격리
- BR-MT01 준수: 모든 주문 조회/생성 시 store_id 필터링
- 다른 매장의 주문에 접근 불가

## 3. 세션 관리 규칙 (BR-SS01 ~ BR-SS03)

### BR-SS01: 세션 자동 시작
- BR-TS01 준수: 해당 테이블에 활성 세션이 없으면 첫 주문 시 자동 시작
- session_id = UUID v4 생성
- session_started_at = 현재 시각
- 이미 세션이 활성 상태이면 기존 session_id 사용

### BR-SS02: 세션 기반 주문 조회
- 현재 테이블의 session_id로 주문 필터링
- session_id가 없으면 (세션 없음) 빈 목록 반환

### BR-SS03: 추가 주문
- 동일 세션 내에서 여러 번 주문 가능
- 각 주문에 별도 order_number 부여
- 모든 주문이 동일 session_id 공유

---

## 4. 주문 성공 후 규칙 (BR-OS01 ~ BR-OS02)

### BR-OS01: 성공 화면 표시
- 주문 성공 시 주문번호, 주문 항목 요약 표시
- 5초 카운트다운 타이머 표시

### BR-OS02: 자동 리다이렉트
- 5초 후 메뉴 페이지로 자동 이동
- 장바구니 초기화 (localStorage 포함)
- 사용자가 타이머 전에 수동으로 이동할 수도 있음

---

## 5. 에러 처리 규칙 (BR-E01 ~ BR-E03)

### BR-E01: 주문 제출 실패 시
- CS-ERR-02 구현: 장바구니 데이터 유지
- 에러 메시지 표시 ("주문 실패. 다시 시도해주세요.")
- 재시도 버튼 제공

### BR-E02: 네트워크 오류
- 서버 응답 없음 시 타임아웃 처리
- 장바구니 데이터 보존
- 에러 토스트 표시

### BR-E03: 유효하지 않은 메뉴 항목
- 서버 검증 시 삭제된 메뉴가 포함되면 422 에러 반환
- 에러 응답에 유효하지 않은 항목 목록 포함
- 클라이언트에서 해당 항목 표시 및 제거 안내
