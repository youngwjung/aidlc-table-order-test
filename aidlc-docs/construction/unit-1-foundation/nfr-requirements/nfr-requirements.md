# 비기능 요구사항 - Unit 1: Foundation

---

## 1. 성능 (Performance)

| 항목 | 목표 | 비고 |
|---|---|---|
| 일반 API 응답 시간 | 500ms 이내 | CRUD 작업 기준 |
| SSE 연결 수립 | 2초 이내 | 초기 연결 기준 |
| SSE 이벤트 전달 | 2초 이내 | 주문 생성~대시보드 표시 |
| 페이지 로드 | 3초 이내 | 초기 로드 기준 |
| DB 쿼리 | 개별 쿼리 200ms 이내 | 인덱스 활용 |

### DB 연결 관리
- SQLAlchemy async 세션 사용
- 연결 풀 크기: 기본값 (5~10)
- 풀 오버플로우: 10

---

## 2. 확장성 (Scalability)

| 항목 | 목표 |
|---|---|
| 동시 접속 테이블 | 10~50개 |
| 동시 주문 | 10~30건 |
| 동시 SSE 연결 | 50개 이하 |
| 매장 수 | 다중 매장 지원 (제한 없음) |

### 확장 전략
- 현재: 단일 서버 (Docker Compose)
- 수평 확장 미고려 (프로토타입)
- SSE: 인메모리 이벤트 큐 (단일 프로세스)

---

## 3. 가용성 (Availability)

| 항목 | 목표 |
|---|---|
| 가용성 목표 | 없음 (프로토타입) |
| 백업 전략 | Docker 볼륨 기반 (수동) |
| 재해 복구 | 미구현 |
| 헬스체크 | 기본 `/health` 엔드포인트 |

---

## 4. 보안 (Security)

| 항목 | 구현 |
|---|---|
| 관리자 비밀번호 | bcrypt 해싱 (salt rounds: 12) |
| 테이블 비밀번호 | 평문 저장 |
| 인증 | JWT (HS256, 16시간 만료) |
| JWT 시크릿 | 환경 변수 관리 |
| 로그인 시도 제한 | 3회 실패 시 1분 잠금 (인메모리) |
| CORS | 프론트엔드 도메인만 허용 |
| SQL Injection | SQLAlchemy ORM (파라미터 바인딩) |
| XSS | Next.js 기본 이스케이핑 |
| HTTPS | 미구현 (프로토타입, Docker 내부 통신) |

---

## 5. 로깅 및 모니터링 (Logging & Monitoring)

| 항목 | 구현 |
|---|---|
| 로깅 수준 | 최소 - 에러만 기록 |
| 로깅 라이브러리 | Python 표준 logging |
| 로그 출력 | stdout (Docker 로그로 수집) |
| 모니터링 | 미구현 |
| 알림 | 미구현 |
| 헬스체크 | `GET /health` → `{"status": "ok"}` |

---

## 6. 데이터 관리 (Data Management)

| 항목 | 구현 |
|---|---|
| DB 마이그레이션 | SQLAlchemy create_all() 직접 (Alembic 미사용) |
| 데이터 백업 | Docker 볼륨 (수동) |
| 이미지 저장 | 로컬 파일 시스템 (Docker 볼륨 마운트) |
| 데이터 보존 | 삭제 정책 없음 (수동 관리) |

---

## 7. 유지보수성 (Maintainability)

| 항목 | 구현 |
|---|---|
| 코드 구조 | 레이어드 아키텍처 (Router → Service → Model) |
| 타입 안전성 | TypeScript (프론트), Pydantic (백엔드) |
| 환경 설정 | .env 파일 + 환경 변수 |
| 문서화 | FastAPI 자동 Swagger UI (`/docs`) |

---

## 8. 사용성 (Usability)

| 항목 | 구현 |
|---|---|
| 터치 타겟 | 최소 44x44px |
| 반응형 | 태블릿 최적화 (고객), 데스크톱/태블릿 (관리자) |
| 에러 피드백 | Toast 메시지 (성공/실패) |
| 로딩 상태 | 버튼 비활성화 + 로딩 인디케이터 |
| 확인 팝업 | 삭제/이용완료 등 파괴적 작업 시 ConfirmModal |
